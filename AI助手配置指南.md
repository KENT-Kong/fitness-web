## 🔧 AI助手工作原理
现在AI助手有两种工作模式：

### 1. 模拟回复模式（当前状态）
- 无需API密钥
- 根据用户提问提供预设的健身建议
- 支持：体重、蛋白质、碳水、饮水、体脂、睡眠、运动等主题

### 2. 完整AI模式（需要配置）
- 需要有效的DeepSeek API密钥
- 使用真正的AI对话功能
- 支持自然语言理解和智能回复

## 🚀 快速启用完整AI功能

### 步骤1：获取DeepSeek API密钥
1. 访问 https://platform.deepseek.com/
2. 注册账号并登录
3. 在API Keys页面创建新密钥
4. 复制API密钥

### 步骤2：配置API密钥
编辑 `fitness-web/.env` 文件：
```env
# Google Sheets API 配置
GOOGLE_SHEETS_API_KEY=YOUR_API_KEY_HERE
GOOGLE_SHEETS_ID=YOUR_SHEET_ID_HERE

# DeepSeek AI API（AI助手功能）
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxx  # ← 替换为你的真实密钥

# 本地开发配置
PORT=8080
HOST=0.0.0.0

# 数据缓存设置（毫秒）
CACHE_DURATION=300000  # 5分钟

# 安全设置
ALLOWED_ORIGINS=http://localhost:8080,http://localhost:3000
```

### 步骤3：重启服务器
```bash
# 停止当前服务器
taskkill /F /IM node.exe

# 启动服务器
cd fitness-web
node server.js
```

## 🐛 常见问题解决

### 问题1：API端点错误
**症状**：前端显示连接失败
**解决**：已修复，前端调用 `/api/ai/chat`

### 问题2：语音功能不工作
**症状**：麦克风或朗读按钮无效
**解决**：
1. 浏览器需要麦克风权限
2. 首次使用时点击允许
3. 确保浏览器支持Web Speech API

### 问题3：服务器启动失败
**症状**：无法访问 http://localhost:8080
**解决**：
1. 检查端口占用：`netstat -ano | findstr :8080`
2. 停止占用进程：`taskkill /F /PID <进程ID>`
3. 重新启动：`node server.js`

### 问题4：AI回复仍然错误
**症状**：显示"网络连接失败"
**解决**：
1. 确认服务器正在运行
2. 检查.env文件配置
3. 尝试刷新页面
